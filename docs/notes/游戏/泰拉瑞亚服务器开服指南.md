# 🌲 泰拉瑞亚 (Terraria) Linux 专用服务器开服指南

> **“为什么我关了电脑，朋友就进不去游戏了？”** —— 因为你缺少一个 24 小时待命的“游戏管家”。
> 本指南将带你从零开始，在 Linux 服务器上搭建一个稳定、持久的泰拉瑞亚世界。

---

## 🏗️ 一、 核心原理：什么是专用服务器？

### 1. 核心原理（一句话解释）
专用服务器（Dedicated Server）是一个**剥离了图形界面的纯逻辑引擎**，它独立运行并同步所有玩家的数据，确保世界的持续存在。

### 2. 生活中的比喻
开服就像是**举办一场永不落幕的桌游派对**：专用服务器就是那位坐在桌子尽头、永远不会疲倦的“游戏裁判”。即使玩家（参加派对的人）回家睡觉了，裁判依然守着棋盘，记录着每一棵树的生长和每一个怪物的刷新。

---

## 🛠️ 二、 为什么要自己开服？ (The "Why")

### 1. 痛点：联机的烦恼
*   **房主依赖**：房主下线，全员退场。
*   **资源占用**：一边玩游戏一边开房，电脑卡得像 PPT。
*   **网络波动**：家庭宽带上传速度受限，队友瞬间瞬移。

### 2. 解决方案：Linux 云服务器
通过在轻量化的 Linux 环境运行服务器程序，可以实现：**低延迟、高可靠、24 小时在线。**

## 🚀 三、 操作步骤：从下载到启航

### 1. 获取服务器程序
```bash
wget https://terraria.org/api/download/pc-dedicated-server/terraria-server-1455.zip
```
> 💡 **为什么要执行这一步**：从官方服务器下载最新的 1.4.5.5 版本服务端压缩包。

```bash
unzip ./terraria-server-1455.zip 
```
> 💡 **为什么要执行这一步**：解压资源包。解压后你会看到 Windows, Mac, Linux 三个平台的文件夹。

```bash
cd ./1455/Linux/
```
> 💡 **为什么要执行这一步**：进入 Linux 专用目录，接下来的操作都在这里进行。

### 2. 赋予执行权限
```bash
chmod +x TerrariaServer.bin.x86_64
```
> ⚠️ **为什么要执行这一步**：Linux 默认禁止运行下载的文件。这一步是把该文件标记为“可执行”，否则你会收到 `Permission denied` 报错。

### 3. 编写配置文件
创建一个名为 `myconfig.txt` 的文件，填入以下内容：
```text
world=/home/wywy521/.local/share/Terraria/Worlds/254.wld
worldname=254
motd=欢迎来到我的泰拉瑞亚服务器！
port=7777
maxplayers=5
password=12345
secure=0
```
> 💡 **为什么要执行这一步**：避免每次开服都要手动输入一堆参数。`world` 指向你的存档路径，`motd` 是进服欢迎语。

### 4. 启动服务器
```bash
sudo ./TerrariaServer.bin.x86_64 -config myconfig.txt
```
> 💡 **为什么要执行这一步**：使用 `sudo` 权限运行程序，并告诉它读取我们刚刚写好的配置文件。

## ✅ 四、 结果验证：如何确认成功？

1.  **控制台心跳**：看到 `Server started` 字样且没有红字报错。
2.  **端口扫描**：在本地电脑使用 `telnet <服务器IP> 7777`，如果连接成功，说明防火墙已开放。
3.  **游戏内连接**：打开泰拉瑞亚客户端 -> 多人游戏 -> 通过 IP 加入 -> 输入 IP 和密码。

---

## ⚠️ 五、 避坑指南 (Critical Tips)

### 1. 防火墙是“头号杀手”
> ⚠️ **常见报错**：连接超时（Connection Timed Out）。
> **解决方案**：请务必在你的服务器后台（腾讯云/阿里云/华为云等）安全组中，开放 **TCP/UDP 7777** 端口。

### 2. 不要直接关闭终端窗口
> 💡 **进阶技巧**：如果你直接关闭 SSH 窗口，服务器进程会被杀掉。请搜索并使用 `screen` 或 `tmux` 命令，让程序在后台持续运行。

### 3. 存档路径权限
> ⚠️ **避坑说明**：如果你的 `world` 路径写在 `/root/` 下，程序可能会因为权限不足无法保存地图，导致“档没了”的悲剧。建议放在 `/home/` 目录下。
