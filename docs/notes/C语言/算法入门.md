## 一、“必须掌握”的基础算法清单（不学这些=算法没入门）

### 1️⃣ 基础数据结构（**第一优先级**）

这是算法的载体，不懂结构，算法无从谈起。

* 数组 / 字符串
* 链表（单向、双向）
* 栈、队列（含循环队列）
* 哈希表（思想即可，重点是“用”）
* 树的基本概念（二叉树、遍历）
* 图的基本概念（不用太深）

> 判断标准：
> 给你结构定义，你能手写增删查改，不依赖 STL。

---

### 2️⃣ 基础算法思想（**核心**）

| 类别        | 必会内容                   |
| --------- | ---------------------- |
| 查找        | 顺序查找、**二分查找（及变体）**     |
| 排序        | 冒泡、选择、插入、**快速排序、归并排序** |
| 递归        | 递归思想、递归转迭代             |
| 分治        | 二分、归并                  |
| 贪心        | 能判断“是否可贪”              |
| BFS / DFS | 图 & 树遍历                |
| 动态规划      | 一维 DP（背包是门槛）           |

> **动态规划不是基础，但“不会=算法不完整”**

---

### 3️⃣ 复杂度分析（不懂这个等于白学）

你至少要能做到：

* 一眼判断时间复杂度
* 知道什么时候会 TLE
* 明白“空间换时间”的代价

## 二、怎么学（正确路径，别走弯路）

### ❌ 错误路线（很多人踩坑）

* 上来刷 LeetCode 中等 / 困难
* 看一堆“算法速成视频”
* 死记模板，不懂为什么

这些**看似努力，实际效率极低**。

---

### ✅ 正确学习路线（工程向 + 可落地）

### **阶段 1：手写 + 理解（关键）**

用 **C** 或 **C++（不许 STL）**：

* 自己写：

  * 二分查找（3 种边界）
  * 快排 / 归并
  * 链表、栈、队列
* 写完必须能解释：

  * 为什么这么写
  * 边界为什么这样处理

> 这个阶段**很痛苦，但决定你上限**

---

### **阶段 2：刷题（少而精）**

LeetCode 不是用来“刷”的，是用来**验证理解的**。

建议：

* 每一类 **5~10 题足够**
* 只刷：

  * Easy
  * 少量 Medium
* 不追求数量，追求**可复现**

---

### **阶段 3：总结模板（工程可用）**

* 自己整理模板（不是抄）
* 模板必须：

  * 可直接用在项目
  * 没有多余魔法代码

---

## 三、书籍推荐（不浪费钱）

### 📘 首选（务实）

1️⃣ **《算法（第4版）》—— Sedgewick**

* 优点：工程思维强
* 缺点：偏厚，但值得
* 适合：你这种偏底层、嵌入式背景

2️⃣ **《数据结构与算法分析（C语言版）》**

* 非常适合嵌入式
* 强调实现细节
* 不花哨

---

### ❌ 不推荐新手

* 《算法导论》
  👉 太理论，容易劝退，不适合你现在

## 五、关键问题：**我是嵌入式，还用学算法吗？**

### 结论一句话：

**要学，但学“够用的算法”，不是竞赛算法。**

### 为什么嵌入式更需要算法？

1️⃣ **资源受限**

* RAM 小
* CPU 慢
* 算法选错，系统直接崩

2️⃣ **实时性**

* O(n) 和 O(log n) 在 MCU 上是生死区别

3️⃣ **驱动 / RTOS / 协议栈**

* 环形缓冲区
* 调度策略
* 内存池
* 查表优化
  👉 全是算法思想

4️⃣ **你不会算法 = 只能当“写业务的嵌入式”**

## 六、给你的务实建议（不鸡汤）

* 不用刷 300 题
* **20~50 道基础题 + 手写实现 = 足够**
* 用 C 写，比用 Python 有价值十倍
* 学算法的目的：

  > **写出可控、可预测、可维护的代码**


| 章节         | C 语言核心知识点            | 对应算法 / 数据结构         | 实践目标                        |
| ---------- | -------------------- | ------------------- | --------------------------- |
| 第 6 章      | 循环（for / while）      | 线性查找（Linear Search） | 能处理空数组 / 单元素数组；写出 O(N) 复杂度  |
| 第 7 章      | 分支（if-else / switch） | 冒泡排序、选择排序           | 用嵌套循环实现 O(N²)，能解释比较与交换次数    |
| 第 8 章      | I/O、基础函数             | 二分查找                | **边界一次写对**，明确左闭右闭或左闭右开      |
| 第 9 章      | 函数、参数传递              | 模块化封装               | 查找 / 排序拆成 `.c/.h`，禁止全写 main |
| 第 10 章（数组） | 一维 / 多维数组            | 快速排序（QuickSort）     | 正确实现 partition，避免死循环        |
| 第 10 章（指针） | 数组与指针                | 归并排序（MergeSort）     | 理解分治 + 辅助数组空间开销             |
| **补充**     | 递归                   | 递归 → 迭代转换           | 能分析递归栈深度，理解嵌入式风险            |
| 第 12 章     | malloc / free        | 动态数组（vector）        | 支持扩容 / 缩容，避免内存泄漏            |
| **补充**     | 内存安全                 | 指针边界检查              | 空指针防护、越界检测                  |
| 第 14 章     | struct               | 链表节点定义              | 正确管理 next 指针生命周期            |
| 第 14 章     | struct + 指针          | 栈 / 队列（链表）          | LIFO / FIFO 行为正确，支持删除       |
| 综合         | 数组 / 指针              | 环形缓冲区               | FIFO 不丢数据、不越界               |
| **综合**     | 复杂度分析                | 算法选型                | 能解释为什么选 O(N) / O(log N)     |
