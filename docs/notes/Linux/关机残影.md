# Arch Linux 关机前清空 TTY 输出并隐藏光标（防残影方案）

在某些笔记本（尤其是 LCD 屏）上，关机时 systemd 的终端输出会在屏幕上停留几秒至几十秒，导致短暂残影。为避免此类现象，可使用自定义 systemd 服务在关机前清空终端并隐藏光标，减轻视觉干扰。

---

## ✦ 原理说明

* **systemd** 默认将关机过程日志输出至 `/dev/tty1`，导致关机前终端内容会“冻结”在屏幕上。
* **LCD 残影** 不是烧屏，但字符停留时间较长时，可能短时间内残留图像，影响体验。
* 通过 systemd 运行脚本向 tty 发送清空命令，并关闭光标，即可避免屏幕长时间显示字符。

---

## ⚙️ 实施步骤（含详细解释）

### 1. 创建 Systemd 服务文件

```bash
sudo vim /etc/systemd/system/clear-tty-on-shutdown.service
```

粘贴以下内容：

```ini
[Unit]
Description=Clear TTY1 and hide cursor before shutdown to avoid image retention
DefaultDependencies=no
Before=shutdown.target reboot.target halt.target

[Service]
Type=oneshot
ExecStart=/usr/bin/bash -c '
  exec > /dev/tty1;
  setterm --cursor off;
  setterm --clear all;
'

[Install]
WantedBy=halt.target reboot.target shutdown.target
```

#### 🔍 每一项说明：

| 字段                       | 作用                   |
| ------------------------ | -------------------- |
| `[Unit]` 段落              | 描述服务属性及启动时机          |
| `DefaultDependencies=no` | 禁用默认依赖，确保尽量靠后运行      |
| `Before=...`             | 表示本服务在真正关机前执行        |
| `[Service]` 段落           | 定义具体执行行为             |
| `Type=oneshot`           | 表示任务只执行一次            |
| `ExecStart=...`          | 实际清空终端和关闭光标的 Bash 脚本 |
| `exec > /dev/tty1`       | 将输出重定向到 `/dev/tty1`  |
| `setterm --cursor off`   | 隐藏光标                 |
| `setterm --clear all`    | 清空终端内容               |
| `[Install]` 段落           | 定义该服务在何种关机目标下启用      |

---

### 2. 重新加载 systemd 配置

```bash
sudo systemctl daemon-reexec
sudo systemctl daemon-reload
```

解释：

* `daemon-reexec`：重启 systemd 本体，确保读取新的配置逻辑。
* `daemon-reload`：重载所有 systemd 单元文件。

---

### 3. 启用服务

```bash
sudo systemctl enable clear-tty-on-shutdown.service
```

解释：

* 创建符号链接，将服务绑定至 `halt.target`、`reboot.target` 和 `shutdown.target`，确保系统无论以哪种方式关机都执行此服务。

---

### 4. 可选：测试服务效果

先手动运行确认无误：

```bash
sudo bash -c 'exec > /dev/tty1; setterm --cursor off; setterm --clear all;'
```

然后切换至 TTY1（按 `Ctrl+Alt+F1`），应看到终端变黑且无光标。

---

## ✅ 成效预期

* 关机时，最后阶段屏幕将被清空，无任何终端字符残留。
* 光标也会被关闭，不再闪烁。
* 屏幕显示为全黑（部分面板为灰黑，取决于背光实现），显著减轻残影风险。

---

## ⚠️ 注意事项

* 该方法适用于 systemd 管理的 Linux 系统，默认 tty 输出为 `/dev/tty1`。
* 若你使用自定义登录管理器或其他 tty，请根据实际调整 `/dev/ttyX`。
* 某些图形环境（如 GDM）会占用 tty1，需将命令输出改至其他 tty（如 `/dev/tty7`）。

---

## 📌 补充建议

* 若屏幕仍残留亮度，可考虑配合使用 `vbetool dpms off` 或 `/sys/class/backlight/` 手动控制背光，但需谨慎，可能在某些系统上导致屏幕无法恢复显示。
