# 连接树莓派指南

## 1. 基本连接方法

### 1.1 查看本机IP
使用以下命令查看本机IP地址：
```bash
ifconfig
```
示例输出：
```
10.42.0.1
```

### 1.2 扫描网络设备
将10.42.0.1替换为自己电脑的IP，使用nmap扫描网络：
```bash
sudo nmap 10.42.0.1/24
```
注：/24表示子网掩码为255.255.255.0

### 1.3 测试连接
使用以下命令测试连接：
```bash
ping raspberrypi.local
```

## 2. 启用mDNS支持

### 2.1 问题描述
Windows 11 能用 `raspberrypi.local` 连接，而 Arch 系统不行，说明问题可能出在 Arch 系统的 mDNS 配置或相关服务上。

### 2.2 安装配置Avahi
1. 安装必要软件包：
```bash
sudo pacman -S avahi nss-mdns
```

2. 编辑NSS配置文件：
```bash
sudo nano /etc/nsswitch.conf
```
确保hosts行包含mdns4_minimal：
```plaintext
hosts: files mymachines myhostname mdns4_minimal [NOTFOUND=return] resolve dns
```

3. 启用并启动服务：
```bash
sudo systemctl enable avahi-daemon
sudo systemctl start avahi-daemon
```

4. 检查服务状态：
```bash
sudo systemctl status avahi-daemon
```

### 2.3 测试mDNS功能
```bash
ping raspberrypi.local
```

## 3. 防火墙配置

### 3.1 检查防火墙状态
```bash
sudo iptables -L
```
或使用ufw：
```bash
sudo ufw status
```

### 3.2 允许mDNS流量
```bash
sudo iptables -A INPUT -p udp --dport 5353 -j ACCEPT
```
或使用ufw：
```bash
sudo ufw allow 5353/udp
```

## 4. 其他排查方法

### 4.1 检查路由器mDNS支持
确保路由器的mDNS/组播功能已启用

### 4.2 清理缓存并重启网络
```bash
sudo systemctl restart avahi-daemon
sudo systemctl restart NetworkManager
```

### 4.3 检查域名解析冲突
```bash
avahi-resolve -n raspberrypi.local
```

## 5. 总结
1. 安装avahi和nss-mdns
2. 配置/etc/nsswitch.conf
3. 启动avahi-daemon服务
4. 检查防火墙设置
