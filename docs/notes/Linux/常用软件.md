# Linux 系统核心软件选型与部署清单

## 1. 背景说明 (Overview)

本清单旨在为 Linux 发行版（以 Arch Linux 及其衍生版为主）提供一套经过工程实践验证的核心软件选型。软件清单涵盖了远程办公、网络诊断、开发工具及系统增强等多个维度。

## 2. 软件部署分类表 (Software Classification)

以下软件基于稳定性与功能性进行选型，安装源优先选择官方仓库（Official Repositories），其次为 Arch 用户仓库（AUR）。

| 软件名称 | 包名 | 软件类型 | 部署来源 | 技术说明 |
| :--- | :--- | :--- | :--- | :--- |
| **Gwenview** | `gwenview` | 图像浏览 | 官方仓库 | KDE 默认的高性能图片查看器，支持基础编辑。 |
| **RustDesk** | `rustdesk` | 远程桌面 | 官方仓库 | 开源的远程连接方案，支持自建服务器（Relay）。 |
| **WiHotspot** | `wihotspot` | 网络共享 | AUR | 具备图形化界面的热点共享工具，基于 `create_ap`。 |
| **Clash (GUI)** | `clash-for-windows-bin` | 网络代理 | AUR | 现代化的跨平台网络代理客户端（带中文补丁版）。 |
| **KHotkeys** | `khotkeys` | 系统增强 | 官方仓库 | KDE 环境下的快捷键与鼠标手势管理核心组件。 |
| **Android Studio** | `android-studio` | 开发工具 | AUR | 针对安卓移动开发的集成开发环境。 |
| **Telegram** | `telegram-desktop` | 即时通讯 | 官方仓库 | 跨平台的高安全性即时通讯客户端。 |
| **Inetutils** | `inetutils` | 网络诊断 | 官方仓库 | 包含 `ftp`、`telnet` 等经典网络基础工具。 |

## 3. 安装与维护建议 (Deployment Strategy)

### 3.1 官方仓库软件安装

对于官方仓库中的包，建议直接通过 `pacman` 进行并行安装：

```bash
sudo pacman -S gwenview rustdesk telegram-desktop inetutils
```

### 3.2 AUR 软件安装规范

> WARNING: AUR 软件由社区维护，安装前请务必核实其 `PKGBUILD` 的安全性与版本更新频率。

推荐使用 `yay` 或 `paru` 作为 AUR 助手进行管理：

```bash
yay -S clash-for-windows-bin android-studio
```

## 4. 常见问题排查 (Troubleshooting)

- **Q: 安装 `clash` 后无法启动？**
  - **A:** 请检查是否由于缺少 `libappindicator` 依赖导致托盘图标无法显示。
- **Q: `inetutils` 中的 `ftp` 指令冲突？**
  - **A:** 若系统中存在多个网络套件，请通过 `ls -l /usr/bin/ftp` 核实符号链接指向。
