# Arch/Manjaro 长时间未更新系统的滚动更新指南

当系统长时间未更新（跨度较大）时，可以通过以下方法安全滚动更新，避免系统挂掉。

## 操作流程

### 1. 更新密钥环

确保系统的 GPG 签名密钥是最新的，先更新 `archlinux-keyring` 和 `archlinuxcn-keyring`。

```bash
sudo pacman -Sy archlinux-keyring archlinuxcn-keyring
```

### 2. 强制刷新包缓存并滚动更新

在更新密钥环后，强制刷新包缓存并执行系统更新：

```bash
sudo pacman -Syyu
```

### 3. 安装 `pacman-static` 以防万一（可选但推荐）

如果更新跨度特别大，为了避免 `pacman` 及其依赖在更新中发生问题，可以提前安装 `pacman-static`：

```bash
sudo pacman -S pacman-static
```

然后使用 `pacman-static` 代替系统自带的 `pacman` 进行更新：

```bash
sudo pacman-static -Syyu
```

这样即使系统库发生变化，更新过程也能正常进行。

### 4. 完成更新后清理系统（可选）

更新完成后，可以清理无用的包缓存和孤立包：

```bash
sudo pacman -Sc
sudo pacman -Rns $(pacman -Qtdq)
```

---

## 注意事项

1. **备份数据**：滚动更新跨度较大时，风险较高，建议提前备份重要数据。
2. **检查镜像源**：确保 `/etc/pacman.d/mirrorlist` 中的镜像源是最新、可用的。
3. **使用 TTY**：在 TTY 环境下执行更新，避免桌面环境可能导致的问题。按 `Ctrl+Alt+F2` 切换到 TTY。
