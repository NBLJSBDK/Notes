# GNU Screen 会话管理指南

## 1. 背景说明 (Background)

`GNU Screen` 是一款功能强大的全屏窗口管理器（终端复用器），它允许用户在单个物理终端或远程 SSH 会话中管理多个虚拟控制台。其核心价值在于支持**会话持久化**：即使网络连接中断，后台运行的任务也不会被终止，用户可以在稍后重新接入会话。

## 2. 核心指令集 (Core Commands)

### 2.1 会话生命周期管理

下表定义了会话管理中最高频的控制命令：

| 操作类型 | 指令格式 | 技术说明 |
| :--- | :--- | :--- |
| **创建会话** | `screen -S <name>` | 创建一个具有指定标签的全新会话。 |
| **列出会话** | `screen -ls` | 查看当前用户下所有已创建的会话及其状态（Detached / Attached）。 |
| **恢复会话** | `screen -r <name/pid>` | 重新接入一个处于后台（Detached）状态的会话。 |
| **强行接入** | `screen -dr <name/pid>` | 若会话在其他终端已被接入，该指令将强制断开远程连接并转接入本地。 |
| **共享会话** | `screen -x <name>` | 允许多个终端同时操作同一个会话，实现多端实时同步显示。 |

> NOTE: 建议在创建会话时使用语义化的 `<name>`，避免在多会话场景下因过度依赖 PID 导致的操作混淆。

### 2.2 复合接入模式（生产推荐）

针对日常快速运维场景，建议使用以下复合参数实现最高接入效率：

- `screen -xRR`: 
  - 优先尝试共享接入（`-x`）已有会话；
  - 若无活跃会话，则尝试恢复（`-R`）断开的会话；
  - 若均不存在，则创建一个全新的默认会话。

## 3. 会话内交互规范 (Interactive Shortcuts)

在 Screen 环境中，所有内置指令均需通过转义组合键 `Ctrl + A` 触发。

| 组合键 | 功能描述 | 备注 |
| :--- | :--- | :--- |
| `Ctrl + A` + `D` | **Detach** (分离会话) | 会话转入后台运行，当前终端返回原始 Shell 界面。 |
| `Ctrl + A` + `C` | **Create** (创建窗口) | 在当前会话中开启一个新的虚拟窗口。 |
| `Ctrl + A` + `N` | **Next** (后一窗口) | 切换至编号递增的下一个虚拟窗口。 |
| `Ctrl + A` + `P` | **Previous** (前一窗口) | 切换至编号递减的上一个虚拟窗口。 |
| `Ctrl + A` + `"` | **Window List** (窗口列表) | 以交互式菜单形式显示当前会话内的所有窗口并进行切换。 |
| `Ctrl + A` + `K` | **Kill** (销毁窗口) | 强制关闭当前的虚拟窗口及其中运行的程序。 |

> WARNING: 执行 `Ctrl + A` + `D` 仅是分离连接，进程将持续运行；而执行 `exit` 或 `Ctrl + D` 则会彻底销毁当前的虚拟窗口。

## 4. 典型应用场景示例

### 任务后台持久化执行
在执行内核编译（`make -j8`）或大规模数据同步等耗时任务时，应遵循以下流程：

1.  启动带标签的会话：`screen -S background_compile`。
2.  执行目标编译指令。
3.  通过 `Ctrl + A` + `D` 分离会话，随后可安全断开 SSH 连接。
4.  通过 `screen -r background_compile` 检查后续执行结果。

## 5. 常见问题排查 (Troubleshooting)

- **Q: 会话显示为 `Attached` 但本地无法连接？**
  - **A:** 使用 `screen -D -r <name>` 强制使远程终端脱离，随后即可在本地重连。
- **Q: 如何彻底清理僵尸会话？**
  - **A:** 进入会话后连续执行 `exit` 直至系统提示 `[screen is terminating]`，或使用 `screen -S <name> -X quit`。
