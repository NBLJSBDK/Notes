# Linux 环境下基于 Xdotool 的自动化文本模拟输入方案

## 1. 背景说明 (Overview)

在日常运维或开发过程中，常需要重复输入复杂的指令或固定的文本内容。通过 `xdotool` 工具，可以模拟键盘按键事件，实现自动化的“宏命令”效果，提升交互效率。

## 2. 环境依赖 (Prerequisites)

> WARNING: 本方案基于 `X11` 显示协议。若当前环境为 `Wayland`（如部分现代 GNOME/KDE 发行版），`xdotool` 可能会失效，建议使用 `ydotool` 作为替代方案。

执行以下指令以核验并安装必要依赖：

```bash
# Arch Linux 环境
sudo pacman -S xdotool
# Ubuntu/Debian 环境
sudo apt install xdotool
```

## 3. 核心实施步骤 (Implementation)

### 3.1 编写自动化脚本

创建一个通用的文本模拟输入脚本（例如 `quick_text_input.sh`）：

```bash
#!/bin/bash
# 延迟执行，留出焦点切换时间（可选）
sleep 0.2

# 模拟键入指定的字符串
xdotool type "hajimi"

# 模拟按下回车键执行
xdotool key Return
```

### 3.2 权限赋予与环境配置

> DANGER: 脚本涉及键盘模拟操作，执行前必须确保脚本权限受控，防止被恶意注入破坏性指令。

```bash
chmod +x ~/scripts/quick_text_input.sh
```

## 4. 交互式快捷键绑定 (Workflow)

为实现“一键输入”，建议将脚本路径绑定至系统全局快捷键：

1.  打开系统设置 -> 键盘 -> 快捷键 (Keyboard Shortcuts)。
2.  添加自定义快捷键 (Custom Shortcut)。
3.  **名称**：`快速文本输入`。
4.  **命令**：`/home/用户名/scripts/quick_text_input.sh`。
5.  **按键绑定**：建议设为 `Super + Alt + K` 等非冲突键位。

## 5. 常见问题排查 (Troubleshooting)

- **Q: 触发后无任何反应？**
  - **A:** 请确认当前终端或窗口是否处于“可输入”激活状态。`xdotool` 仅向当前获取焦点的窗口发送指令。
- **Q: 中文字符模拟失败？**
  - **A:** `xdotool type` 原生对中文字符集支持有限。建议改用 `xclip` 将内容复制到剪贴板，再由 `xdotool` 模拟 `Ctrl + V` 粘贴。
