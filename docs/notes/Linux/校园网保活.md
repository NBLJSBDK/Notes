可以直接用 **systemd 定时器**，比 cron 更适合 Arch（Arch 默认没有启用 cron）。下面给出最干净、可靠的方案。

---

## 一、先写一个脚本（代替 bat）

在你的 Arch 上创建脚本：

**/usr/local/bin/keepalive.sh**

```bash
#!/bin/bash
curl "http://<login_page_ip>/a70.htm" -d "DDDDD=<school_id>&upass=<school_pass>&0MKKey="
```

然后赋予权限：

```bash
sudo chmod +x /usr/local/bin/keepalive.sh
```

---

## 二、创建 systemd service

**/etc/systemd/system/keepalive.service**

```ini
[Unit]
Description=Keepalive curl script

[Service]
Type=oneshot
ExecStart=/usr/local/bin/keepalive.sh
```

---

## 三、创建 systemd timer（每 8 分钟执行一次）

**/etc/systemd/system/keepalive.timer**

```ini
[Unit]
Description=Run keepalive script every 8 minutes

[Timer]
OnBootSec=1min
OnUnitActiveSec=8min
Unit=keepalive.service

[Install]
WantedBy=timers.target
```

---

## 四、启用并启动定时器

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now keepalive.timer
```

查看运行情况：

```bash
systemctl status keepalive.timer
```

查看任务是否定时执行：

```bash
journalctl -u keepalive.service -f
```

---

## 这样就能稳定地每 8 分钟自动执行一次 curl 防掉线

无需 Cron，Arch 上用 systemd timer 是最佳方式。
