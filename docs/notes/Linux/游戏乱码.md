
# Steam 启动游戏乱码问题解决方案（Arch Linux）

## 🎯 问题背景
在 Arch Linux 下启动某些游戏（如 Shattered Pixel Dungeon）时，遇到 **中文乱码或文本错乱** 的问题。

---

## 🛠️ 解决方案：使用 wrapper 脚本包装原始启动器

### 原始情况
游戏启动文件名为：
```
Shattered Pixel Dungeon   # 无文件后缀
```

### 步骤如下：

#### 1. **重命名原始启动文件**
 ```bash
mv "Shattered Pixel Dungeon" "Shattered Pixel Dungeon.real"
```

#### 2. **新建一个 wrapper 脚本（同名）**

```bash
   touch "Shattered Pixel Dungeon"
   chmod +x "Shattered Pixel Dungeon"
```

#### 3. **编辑新脚本内容**

```bash
   #!/bin/bash
   export LANG=en_US.UTF-8
   export LC_ALL=en_US.UTF-8
   exec "$(dirname "$0")/Shattered Pixel Dungeon.real" "$@"
```

---

## ✅ 更优方式：直接在 Steam 启动选项中设置环境变量

不修改游戏文件，直接在 **Steam → 游戏属性 → 启动选项** 中添加：

```bash
LANG=en_US.UTF-8 LC_ALL=en_US.UTF-8 %command%
```

这样 Steam 启动游戏时会自动设置编码，避免乱码问题。

---

## ❓ 为什么中文环境会导致问题？

* `zh_CN.UTF-8` 可能触发游戏或系统使用中文字体/字符集，但游戏或库不支持，导致乱码。
* 某些游戏依赖 ASCII 环境，如果 locale 设置为中文，会错误解析字符。
* `en_US.UTF-8` 强制使用标准 UTF-8 编码，避免语言相关处理逻辑，反而更稳定。

---

## 🧠 可选替代方案（保留中文环境）：

```bash
LANG=zh_CN.UTF-8 LC_CTYPE=en_US.UTF-8 %command%
```

这样能保持中文界面，但用英文编码避免乱码。适合特定需求下使用。

---

## ✅ 最终结果

成功解决乱码问题，游戏可正常运行，中文或其他语言内容不会出现文本错乱。
