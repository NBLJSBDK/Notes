以下是我为你整理的 VMware 双网卡环境配置指南（用于上网 + 局域网 SSH），按照顺序操作即可：

---

## ✅ 一、VMware 虚拟网络设置

在 **VMware Workstation → Edit → Virtual Network Editor** 中进行设置，点击 **"Change Settings"** 获取管理员权限。

### 1. **配置三种虚拟网络**

| 网络名    | 类型        | 用途         | 子网设置                                                                          |
| ------ | --------- | ---------- | ----------------------------------------------------------------------------- |
| VMnet8 | NAT       | 上网         | Subnet IP: `192.168.88.0`<br>DHCP 开启：IP范围建议 `192.168.88.128 ~ 192.168.88.254` |
| VMnet0 | Bridged   | 局域网通信（SSH） | **桥接到宿主机物理网卡（手动选择）**，不要选“自动”                                                  |
| VMnet1 | Host-only | 可选，仅主机通信   | Subnet IP: `192.168.56.0`（不要留空），可启用 DHCP                                      |

---

## 🖥️ 二、配置虚拟机网卡

在虚拟机关机状态下操作：

### 1. 添加两个网卡：

* 网卡 1：**NAT** 模式，连接到 VMnet8 → 用于上网
* 网卡 2：**桥接** 模式，连接到 VMnet0 → 用于局域网（SSH）

### 2. 确保：

* 勾选每张网卡的：

  * `Connected`
  * `Connect at power on`

---

## 🧱 三、虚拟机内 Linux 网络配置（开机后）

### 1. 查看当前网卡名

```bash
ip a
```

你应看到两块网卡，例如 `ens33` 和 `ens36`（名称可能不同）。

* NAT 网卡一般是 `ens33`（VMnet8）
* 桥接网卡一般是 `ens36`（VMnet0）

可用以下方式确认哪个是哪个：

```bash
ethtool ens33
```

看 Link 状态和是否能通过 DHCP 拿 IP 来判断。

---

### 2. 获取 NAT 网卡（ens33）IP 上网：

```bash
sudo dhclient ens33
ping 223.5.5.5
ping baidu.com
```

---

### 3. 获取桥接网卡（ens36）局域网 IP：

```bash
sudo dhclient ens36
ip a
```

应看到局域网 IP，例如：`192.168.1.x`

---

### 4. 启动 SSH 服务

```bash
sudo systemctl enable sshd --now
```

用宿主机连接：

```bash
ssh 用户名@192.168.1.x
```

---

## 🧪 最终验证 checklist

| 功能       | 命令                     | 预期结果                |
| -------- | ---------------------- | ------------------- |
| 能上网      | `ping baidu.com`       | 能 ping 通            |
| 有局域网 IP  | `ip a`                 | 显示 `192.168.x.x` 地址 |
| 能 SSH 登录 | `ssh user@192.168.x.x` | 成功连接                |

## 📌 附：VMnet 子网设置要求（不可留空）

| VMnet编号 | 模式        | 是否允许空 | 备注说明                        |
| ------- | --------- | ----- | --------------------------- |
| VMnet8  | NAT       | ❌ 不可  | 必须设置子网 IP，例如 `192.168.88.0` |
| VMnet0  | Bridged   | ✅ 可   | 无需设置子网，由真实路由器分配             |
| VMnet1  | Host-only | ❌ 不可  | 建议设置子网 IP，例如 `192.168.56.0` |

> VMnet0 建议手动指定桥接的物理网卡，不要选“自动”。

## 如需静态 IP 配置，请告知：

如果你希望设置静态 IP（如 `192.168.88.10`），可选择：

* `netplan`
* `systemd-networkd`
* 或传统 `/etc/network/interfaces`
