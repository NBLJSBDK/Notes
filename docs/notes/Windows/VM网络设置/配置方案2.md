很好，现在你已启动虚拟机，我们继续进行 **网络配置**。

---

## ✅ 目标回顾：

* `ens33` → NAT 模式（连接 VMnet8，用于上网）
* `ens36` → 桥接模式（连接 VMnet0，用于局域网/SSH）

---

## 🧭 第一步：确认网卡设备名

先确认哪张是 NAT，哪张是桥接：

```bash
ip a
```

输出中你会看到 `ens33`、`ens36`、或者 `eth0`、`eth1` 等。

你也可以用下面的命令判断连接状态：

```bash
ethtool ens33
ethtool ens36
```

输出中 `Link detected: yes` 的就是插着“网线”的网卡（即虚拟网络连接的那张），一般：

* `ens33` 是 NAT（VMnet8）
* `ens36` 是桥接（VMnet0）

---

## 🛠 第二步：让虚拟机连上 NAT 网络（先保证能上网）

执行以下命令：

```bash
sudo dhclient ens33
```

然后验证是否拿到 IP：

```bash
ip a
```

你应该能看到 `ens33` 获得了 IP，比如：

```
inet 192.168.88.130/24 brd 192.168.88.255 ...
```

再验证网络连通：

```bash
ping 223.5.5.5    # 测试外网IP
ping baidu.com    # 测试DNS解析
```

---

## 🧰 第三步：配置桥接网卡（局域网通信）

### 方式一：自动获取（推荐）

```bash
sudo dhclient ens36
```

然后查看：

```bash
ip a
```

如果你家路由器启用了 DHCP，虚拟机会自动获得类似 `192.168.1.x` 这种地址。此时你可以从宿主机或其他局域网设备：

```bash
ssh 用户名@192.168.1.x
```

### 方式二：手动配置静态 IP（如果没有路由器 DHCP）

编辑配置文件（假设使用 `netplan`）：

```bash
sudo nano /etc/netplan/01-bridge.yaml
```

内容如下（根据你网卡名调整）：

```yaml
network:
  version: 2
  ethernets:
    ens36:
      addresses:
        - 192.168.1.100/24
      gateway4: 192.168.1.1
      nameservers:
        addresses: [223.5.5.5, 8.8.8.8]
```

应用配置：

```bash
sudo netplan apply
```

---

## 🧪 第四步：验证 SSH 登录

1. 确保 SSH 服务运行：

```bash
sudo systemctl enable sshd --now
```

2. 从宿主机终端登录：

```bash
ssh 用户名@192.168.1.100
```

---

## 🧼 补充建议

* 如果你不需要 `virbr0` 这类 KVM 的虚拟网桥，可以禁用掉以减少干扰。
* 可以将网卡配置写入系统服务，避免每次开机都 `dhclient`。
