# 我现在还原了虚拟网络设置 虚拟机关机了 请你帮我配置

很好，虚拟机已关机且你已重置虚拟网络，我们现在从头一步步配置，目标是：

1. 一张网卡用于上网（NAT 模式）
2. 另一张网卡用于局域网通信（SSH 进虚拟机）

---

## 🧱 一、VMware 虚拟网络配置

### 1. 打开 VMware Virtual Network Editor

在宿主机 VMware Workstation 中：

* 顶部菜单栏点击 `Edit` → `Virtual Network Editor`
* 需要点击下方按钮 **“Change Settings”** 进入管理员权限模式

确保下面三种虚拟网络存在（若没有就点击 Add 添加）：

| 名称     | 类型            | 用途说明        |
| ------ | ------------- | ----------- |
| VMnet0 | Bridged       | 局域网通信用（SSH） |
| VMnet8 | NAT           | 上网用         |
| VMnet1 | Host-only（可选） | 如果你只想从宿主机访问 |

确保：

* VMnet8 是 NAT 模式，**启用 DHCP 服务**
* VMnet0 是桥接模式，桥接你的物理网卡（比如 `Wi-Fi` 或 `以太网`）

---

## 🧰 二、虚拟机添加两个网卡

### 1. 进入虚拟机设置（虚拟机关机状态）

* `VM` → `Settings` → `Add...` → 添加 `Network Adapter`，添加 **两个网卡**

  * 网卡1：选择 `NAT`（连接到 VMnet8，用于上网）
  * 网卡2：选择 `Bridged`（连接到 VMnet0，用于局域网 SSH）

确保：

* 勾选两个网卡的 `Connected` 和 `Connect at power on`

---

## 🚀 三、启动虚拟机后配置网络

登录虚拟机（Linux），执行以下步骤。

### 1. 查看网卡名称

```bash
ip a
```

你会看到两个网卡，比如 `ens33` 和 `ens36`，通常：

* `ens33` 是 NAT（VMnet8）
* `ens36` 是桥接（VMnet0）

（若名称不一致，用 `ethtool ens33` 判断是否连接，或看 IP 范围）

---

### 2. NAT 网卡上网配置（ens33）

```bash
sudo dhclient ens33
```

验证：

```bash
ping 223.5.5.5
ping baidu.com
```

如果能 ping 通就说明上网正常。

---

### 3. 桥接网卡配置（ens36）

假设你用桥接模式打算从路由器分配局域网 IP，你可以也使用 DHCP：

```bash
sudo dhclient ens36
```

查看 IP：

```bash
ip a
```

你应该能看到一个局域网 IP，例如 `192.168.1.100`

---

### 4. 启动 SSH 服务（确保你能远程连接）

```bash
sudo systemctl enable sshd --now
```

从你的宿主机使用：

```bash
ssh 用户名@虚拟机IP
```

---

# VMnet8 是 NAT 模式  底下的子网ip空着吗


**不可以空着**，`VMnet8` 是 NAT 网络，必须配置子网 IP 段，否则虚拟机无法通过 NAT 获取 IP 地址，也无法上网。

---

## ✅ 正确设置方式如下：

在 VMware 的 **Virtual Network Editor** 中：

### 找到 `VMnet8`（类型应为 NAT），然后设置：

| 选项                       | 设置示例                  |
| ------------------------ | --------------------- |
| **Subnet IP**            | `192.168.88.0`        |
| **Subnet mask**          | 自动填充为 `255.255.255.0` |
| 勾选 `Use local DHCP...`   | ✅（必须勾选）               |
| 点击右边的 `DHCP Settings...` | 查看分配范围设置              |

### DHCP 范围（点击 `DHCP Settings...`）推荐设置：

| 项目       | 建议值              |
| -------- | ---------------- |
| Start IP | `192.168.88.128` |
| End IP   | `192.168.88.254` |

说明：这样虚拟机通过 DHCP 能自动获取到 `192.168.88.x` 范围内的 IP。

---

## 🛠 设置完成后，点击 “Apply” 保存，再启动虚拟机。

虚拟机内执行：

```bash
sudo dhclient ens33
```

然后查看 IP：

```bash
ip a
```

确认是否获取到 `192.168.88.x` 的地址。

---

## ⚠️ 提醒

如果你设置完子网后还是无法上网：

* 重启 VMware 的虚拟网络服务（或重启电脑）
* 确保虚拟机网络适配器绑定的是 `VMnet8`，而不是“自动”

## ❓1. **VMnet0 桥接选择 “已桥接至：自动” 吗？**

**不建议选择“自动”**，建议你手动选择你要桥接的**物理网卡接口**，原因如下：

### ✅ 推荐做法：

在 **VMnet0（桥接模式）** 的设置中：

* 取消勾选 “桥接至自动”
* 手动选择你当前联网的物理网卡，例如：

| 场景    | 应选物理网卡                     |
| ----- | -------------------------- |
| 使用有线网 | `Intel Ethernet` 等名        |
| 使用无线网 | `Intel Wi-Fi`, `Realtek` 等 |

> 🔧 手动绑定可以避免 VMware 错误选择未连接或虚拟的网卡，尤其你使用的是笔记本或有多个接口的设备。

---

## ❓2. **VMnet1（仅主机模式）的子网 IP 可以空着吗？**

**不能空着**，必须设置一个子网 IP 段。否则虚拟机和宿主机将无法通过仅主机模式通信。

### ✅ 推荐设置方式如下：

| 项目          | 建议值                 |
| ----------- | ------------------- |
| Subnet IP   | `192.168.56.0`      |
| Subnet Mask | 自动填 `255.255.255.0` |
| 是否启用 DHCP   | 可选，视是否自动分配          |

如果你希望虚拟机使用固定 IP，不启用 DHCP 也可以，关键是：

* 宿主机虚拟网卡 `VMnet1` 会获取例如 `192.168.56.1`
* 你可以在虚拟机中静态配置 IP 为 `192.168.56.100`，网关设为 `192.168.56.1`

---

## ✅ 建议配置总结：

| VMnet编号 | 模式        | 用途      | 子网IP示例         | 是否启用DHCP |
| ------- | --------- | ------- | -------------- | -------- |
| VMnet0  | 桥接        | 局域网/SSH | 无需设置           | 无需设置     |
| VMnet8  | NAT       | 上网      | `192.168.88.0` | ✅        |
| VMnet1  | Host-only | 宿主机通信   | `192.168.56.0` | 可选（推荐）   |
